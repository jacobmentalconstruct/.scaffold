# `config/tool_manifest.json` — PROSE PLAN

> **Status:** Tranche B plan. Actual JSON is auto-generated by `tool_registry_manager` on tool register/unregister. This file describes the shape.

## Purpose

A **machine-readable index of every tool** registered under `src/tools/`. Companion to the human-facing [_docs/reference/TOOLS.md](../_docs/reference/TOOLS.md). Adapted from the precursor's `tool_manifest.json` pattern (see `_docs/migration/INCORPORATION_INVENTORY.md` §5.3).

The agent uses this file to know what tools exist, what arguments they take, what authority they need, and how to call them.

## Shape (planned)

```jsonc
{
  "manifest_version": "1.0",
  "generated_at": "ISO 8601 UTC",
  "sidecar_id": "<from journal_meta>",
  "tool_count": <int>,

  "tools": [
    {
      "tool_name": "journal_write",
      "version": "1.0.0",
      "entrypoint": "src/tools/journal_write.py",
      "category": "write",
      "summary": "Create, update, append journal entries.",
      "mcp_name": "journal_write",
      "required_authority": "Propose",
      "input_schema": { /* JSON Schema */ },
      "result_envelope_keys": ["status", "tool", "input", "result"],
      "registered_at": "ISO 8601 UTC",
      "source_hash": "sha256:..."
    }
    // ... one entry per tool
  ],

  "categories": {
    "bootstrap": ["sidecar_install", "journal_init", "journal_acknowledge"],
    "scan": ["file_tree_snapshot", "workspace_boundary_audit", ...],
    "write": ["journal_write", "text_file_writer"],
    "query": ["journal_query", "text_file_reader", "repo_search", "read_projection"],
    "scaffold": ["directory_scaffold"],
    "memory": ["session_evidence_store"],
    "testing": ["text_file_validator"],
    "security": ["secret_surface_audit"]
  }
}
```

## Field meanings

- **`tool_name`** — unique identifier, matches `FILE_METADATA["tool_name"]`.
- **`version`** — semver from `FILE_METADATA["version"]`.
- **`entrypoint`** — file path relative to sidecar root.
- **`category`** — see categories list in [_docs/reference/TOOLS.md](../_docs/reference/TOOLS.md).
- **`summary`** — one-liner (matches `FILE_METADATA["summary"]`).
- **`mcp_name`** — the name used to register with the MCP server.
- **`required_authority`** — minimum authority level to invoke this tool. The ContractAuthority enforces this.
- **`input_schema`** — JSON Schema for the tool's `arguments`.
- **`result_envelope_keys`** — should always be `["status", "tool", "input", "result"]` per the Standard Tool Contract; included for clients that want to validate.
- **`registered_at`** — when this tool was discovered and registered.
- **`source_hash`** — SHA-256 of the tool's source file. Used to detect drift; if a tool's source changes, the manifest is regenerated.

## Generation discipline

- Auto-generated by `tool_registry_manager`:
  - On install (initial discovery).
  - On every `register_tool` / `unregister_tool` envelope.
  - On `tool_registry_manager` boot if `source_hash` mismatch detected for any tool.
- File is derived state — regeneratable from the `tool_registry` table in the DB.

## Consumers

- **MCP clients (agents)** — to know what tools to call and with what input.
- **CLI users** — `python -m src.app cli list-tools` reads this.
- **External tooling** — for tool discovery from outside the sidecar.

## Non-goals

- Does not include tool source code (the `source_hash` is enough for integrity checks).
- Does not include runtime metrics (that's a future projection).
- Does not duplicate `toolbox_manifest.json` — this is just the per-tool detail layer.

## Open questions for code time

- Whether to include `input_schema` inline (could make the file large) or reference it (e.g., `input_schema_ref: "blob:sha256/..."`). Lean inline at MVP; revisit if size becomes painful.
- Whether categories are an enum (defined in `src/schemas/`) or free-form (any string a tool's `FILE_METADATA["category"]` declares). Lean enum to prevent drift; document in `src/schemas/`.
